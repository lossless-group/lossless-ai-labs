## Why Care?

The `maintain-splash-pages` skill specifies a `<SortControls>` component on the `/changelog/` and `/context-v/` list pages: four sort keys (Modified, Created, Published, Title), per-page `localStorage` persistence, server-pre-sorted default so the static HTML matches the UI default before JS hydrates. The ai-labs splash had a fixed server-side sort and no way for a reader to flip the order. This entry closes that gap, lifted from `lfm/splash` with the localStorage namespace adapted.

After this lands, the only remaining skill gap is the `/llms.txt` + `/llms-full.txt` endpoints — a mechanical port from the two children that already ship them.

## What's New?

- **`splash/src/components/SortControls.astro`** — chip-group radio for the four sort keys + a direction toggle that adapts its label to value type (dates flip "Newest first / Oldest first", titles flip "A → Z / Z → A"). Adapted from `lfm/splash` with one change: `localStorage` namespace is `ai-labs-splash-sort:<name>` (matching the splash's existing `ai-labs-splash-mode` mode-toggle convention).
- **`changelog/index.astro`** carries `<SortControls name="changelog" ...>` above its `<ul>`. The `<ul>` is annotated `data-sort-target="changelog"`. Each `<li>` carries `data-sort-modified`, `data-sort-created`, `data-sort-published`, `data-sort-title` — ISO date strings for the three date keys (lex-sort works as time-order), lowercased text for title. Empty values sink to the bottom regardless of direction.
- **`context-v/index.astro`** carries one `<SortControls name="context-v" ...>` above all five section blocks (`specs`, `explorations`, `plans`, `prompts`, `reminders`). Each section's `<ul>` carries `data-sort-target="context-v"`. The component's JS finds all five lists on click and sorts each independently — so when you sort by title, each section's entries reorder alphabetically *within that section* and the visual section-grouping stays intact. That's actually a better UX than collapsing everything into one big list.
- **Server pre-sort** matches the SortControls default (`modified` desc) so the page reads correctly with JS off or before hydration. Once JS loads, the component restores any persisted choice from `localStorage`.

## How the data flow works

```
                  ┌─────────────────────────┐
                  │  /context-v/  page      │
                  └────────┬────────────────┘
                           │
              ┌────────────┼────────────┐
              │            │            │
              ▼            ▼            ▼
    SortControls       counts strip    section blocks
    (one)                              (five <ul>s, all data-sort-target="context-v")
              │
              │  user clicks "Title"
              │  user toggles "A → Z"
              ▼
    document.querySelectorAll('[data-sort-target="context-v"]')
              │
              ▼
    five <ul>s reorder independently
    each <li> sorted by data-sort-title lex
              │
              ▼
    localStorage.setItem('ai-labs-splash-sort:context-v', {key:'title', direction:'asc'})
```

Next page-load: the JS reads the persisted state, repaints the chip-group + arrow label, and re-runs `sortAll()` if the persisted state differs from the server-default.
