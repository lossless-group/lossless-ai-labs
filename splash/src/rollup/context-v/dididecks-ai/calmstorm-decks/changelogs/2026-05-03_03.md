## Why Care?

The first time a stakeholder pastes one of these URLs into Slack, an email
preview, an iMessage thread, a LinkedIn DM, a calendar invite, or a Notion
page, the link unfurls into a card. That card is the first impression of
the entire deck before anyone clicks anything. If the card is generic
("Astro" / a blank box) the link looks unfinished. If the card carries the
Calm/Storm wordmark with a brand-aligned image and a sentence describing
*this specific page*, the link reads as a deliberate piece of work.

That is the entire game with OpenGraph metadata: it controls the impression
that precedes the first click. It costs nothing at runtime, it is invisible
on the page itself, and it is one of the few places where a small amount of
discipline carries forward through every share of every URL — forever, at
zero per-share cost.

Beyond social previews, the same metadata block does three other useful jobs:

1. **Browser titles** become precise instead of generic — every tab in a
   reviewer's browser tells them what they are looking at.
2. **Canonical URLs** lock down the absolute address each page is meant to
   live at, which matters the moment the deck is shared across stakeholders
   running different preview environments.
3. **Robots directives** declare unambiguously to indexers — Google, Bing,
   archival bots, AI crawlers — that this content is not for public
   discovery, doubling-up the protection set in `robots.txt` and the
   `X-Robots-Tag` header from earlier this week.

The system shipped today is built so that **adding a new gated page is a
single entry in a registry, and the entire metadata posture follows
automatically**. As the workspace continues to grow — new slides, new
scroll variants, new explorations — the social-preview surface scales with
it instead of needing per-page reinvention.

## What Was Built

### A single source of truth for page metadata

`src/lib/seo.ts` is the registry every page reads from. It exports:

- **Site-wide constants**: `SITE_NAME`, `SITE_TAGLINE`, `DEFAULT_OG_IMAGE`
  (the new `og_image__CalmStorm-Deck-Fund-III.png` shipped to `/public`),
  the image's exact pixel dimensions (3064×1856) and alt text, and a
  standardized `TITLE_SUFFIX` ("Calm/Storm") that gets appended to every
  page title for consistent browser-tab framing.
- **`SLIDE_SEO`** — a 17-entry registry, one per slide slug, holding a
  `title` and `description` for each. Every slug in
  `src/lib/slides.ts` has a corresponding entry written from the slide's
  narrative content.
- **`SCROLL_DECK_SEO`** — keyed by route (`/thesis`, `/thesis/version-2`,
  `/thesis/version-3`), so each scroll variant carries its own preview
  copy describing what the variant is.
- **`STATIC_SEO`** — entries for one-off pages: the root cover, the
  changelog index, plus a `variantChooser()` factory function for the
  per-slide choosers.
- **Helpers**: `getSlideSeo(slug)`, `getScrollDeckSeo(pathname)`, and
  `buildPageTitle(title)` for composing the suffixed `<title>`.

The intent: page templates should never inline title or description
strings. They look the entry up by slug or path, or pass the registry's
exported constant to the metadata component. This means changing how the
deck describes itself anywhere is a single-file edit.

### A comprehensive metadata component

`src/components/basics/MetaTags.astro` is the one component every page
drops into `<head>`. It takes minimum props (`title`, `description`) and
emits the full block:

- `<title>` (with the suffix appended)
- `<meta name="description">`
- `<meta name="robots">` (defaults to `noindex, nofollow, noarchive,
  nosnippet`; opt out with `noindex={false}`)
- `<link rel="canonical">` (resolved to an absolute URL using
  `Astro.site` and `Astro.url.pathname`)
- **OpenGraph** — `og:type`, `og:site_name`, `og:title`, `og:description`,
  `og:url`, `og:image`, `og:image:secure_url`, `og:image:type`,
  `og:image:width`, `og:image:height`, `og:image:alt`
- **Twitter / X** — `twitter:card="summary_large_image"`,
  `twitter:title`, `twitter:description`, `twitter:image`,
  `twitter:image:alt`

Overrides available on every page through props: `ogImage`, `ogImageWidth`,
`ogImageHeight`, `ogImageAlt`, `ogType` (e.g. `"article"`), `noindex`, and
`canonical`. The default OG image is the one shipped today and used by
every page; per-page overrides are a single attribute when a custom image
gets generated for a particular surface.

### Absolute-URL resolution

`astro.config.mjs` now sets a `site` value:

```js
site: process.env.SITE_URL ?? "https://calmcloud-decks-mso9.vercel.app",
```

This is what turns relative paths into absolute URLs in the OG output —
critical, because Slack, X, LinkedIn, and most preview unfurlers refuse
relative `og:image` paths. When the deck moves off the Vercel preview
domain, set `SITE_URL` in Vercel's env panel and every emitted URL
updates automatically.

### Wired through every user-facing surface

| Surface | Source of metadata |
|---|---|
| `/` (cover + menu) | `STATIC_SEO.root` |
| `/thesis` (v1) | `getScrollDeckSeo("/thesis")` |
| `/thesis/version-2` (v2) | `getScrollDeckSeo("/thesis/version-2")` |
| `/thesis/version-3` (v3) | `getScrollDeckSeo("/thesis/version-3")` |
| `/{slug}` (4 slide choosers today) | `getSlideSeo(slug)` |
| `/drafts/{slug}/{slug}-vN` (every draft variant) | `getSlideSeo(slug)` via `SlideLayout` |
| `/changelog` | `STATIC_SEO.changelogIndex` |
| `/changelog/[slug]` | Each entry's frontmatter `title` + `lede`; `og:type="article"` |

`SlideLayout.astro` is the single point that wires every slide-by-slide
draft (and every per-slug chooser that uses it) — so the four chooser
pages and the ~21 draft pages all picked up the new metadata with no
per-page touches beyond two lines in the layout itself.

### Branded shared OG image

`/public/og_image__CalmStorm-Deck-Fund-III.png` (3064×1856 PNG, 256 KB) is
the single branded social-preview image used by every page today. It
mirrors the cover slide of the deck — wordmark, "Europe's #1 HealthTech
Fund" subhead, "Strictly Confidential" annunciator, EuVECA classification,
disclaimer band, "Fund III · Teaser" stamp, copyright. Anyone who unfurls
any deck URL sees this card.

The architecture deliberately leaves room for **per-page custom OG images
in the future** — pass `ogImage="/some-other.png"` (and matching width /
height / alt) to `<MetaTags>` and the component handles absolute-URL
resolution automatically.

## Open Items

- **Per-page custom OG images.** v1 ships with one shared image across
  every surface, which is the right baseline for a small workspace where
  every URL leads to the same overall artifact. Specific pages might
  benefit from custom images later — a unique card for the Numbers
  spread, for the Portfolio Snapshot, for major changelog entries.
  Architecture supports it today; we just haven't generated the assets.
- **Production `SITE_URL` env var.** Once the deck moves off the current
  Vercel preview domain to its eventual production URL, set `SITE_URL` in
  Vercel's project settings. Until then, the canonical/OG URLs will read
  as `https://calmcloud-decks-mso9.vercel.app/...`, which is correct for
  the preview environment.
- **Slide-narrative metadata in two places.** Each slide's title /
  description now lives in `src/lib/seo.ts`; the slide's longer narrative
  lives in `context-v/narratives/{n}-{slug}.md`. Worth a future refactor
  to derive `SLIDE_SEO` automatically from the narrative frontmatter
  (the narratives already carry `eyebrow`, `headline`, and `subhead`).
  Deferred until a slide's preview copy diverges meaningfully from its
  narrative summary.
- **`og:type="article"` only on changelog detail.** Every other page
  defaults to `og:type="website"`. If we add a true blog or longform
  documentation later, those pages should opt into `article` for richer
  unfurl behavior on platforms that distinguish.
- **Image preflight.** Some platforms cap OG images at 5MB and 1200px
  width. Today's image is 256KB and 3064px wide — well under the size
  limit but wider than some platforms' ideal. Most unfurlers downsample
  cleanly; if any platform misbehaves, generating a 1200×727 sibling
  is straightforward.
