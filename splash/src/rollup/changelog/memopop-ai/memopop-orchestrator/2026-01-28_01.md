## New Module: Research PDF Scraper

A complete standalone module for parsing market research PDFs (McKinsey, CB Insights, PitchBook, Gartner, etc.) into markdown with properly formatted Obsidian-style citations.

### Design Decision: Scraper vs Agent

The module distinguishes between:
- **Scraper** (deterministic): `src/scrapers/research_pdf/` - No LLM calls, pattern-based extraction
- **Agent** (LLM-driven): `src/agents/citation_corrector.py` - Uses Claude for correction and recovery

This separation allows:
1. Fast, cost-free PDF parsing for most cases
2. Optional LLM enhancement when higher accuracy is needed
3. Clear separation of concerns

## Citation Format Conversion

### Input Formats Detected

The scraper handles multiple citation styles:

| Format | Example | Detection Pattern |
|--------|---------|-------------------|
| LaTeX inline | `markets.1 2 3` followed by `1U.S. Food and Drug...` | `^\d{1,3}[A-Z]` |
| Superscript | `text¹²³` | Unicode superscript chars |
| Bracketed | `[1], [2,3], [1-5]` | `\[\d+\]` pattern |
| Endnotes | `1. Author, "Title"...` | Numbered list in refs section |

### Output Format (Obsidian-style)

**Inline references:**
```markdown
Text with claim. [^10] [^11] [^12] More text continues here.
```

**Reference definitions:**
```markdown
[^10]: 2018. [Chronic nicotinamide riboside supplementation...](). Published: 2018 | Updated: N/A
```

### Spacing Rules (Critical)

Per `context-v/reminders/Extended-Markdown-Citation-System-Syntax.md`:

- **One space after punctuation** before citation: `text. [^1]`
- **One space between** multiple citations: `[^1] [^2] [^3]`
- **One space after** last citation before text: `[^3] Text continues`
- **No space before** opening bracket on definition: `\n[^1]: ...`

### 2. LaTeX Footnote Detection (`citation_scraper.py`)

The key innovation for handling LaTeX-generated PDFs where footnotes appear inline:

```python
def extract_latex_footnotes(self, text: str) -> List[Dict[str, Any]]:
    """
    Extract footnotes from LaTeX-generated PDFs where footnotes are
    embedded inline in the extracted text.

    LaTeX footnotes appear as: "1U.S. Food and Drug Administration..."
    where the number is immediately followed by the citation text.
    """
    # Pattern: number at start of line followed by capital letter
    # Examples:
    #   1U.S. Food and Drug Administration, "FDA Approves..."
    #   10Martens CR, et al., "Chronic nicotinamide..."

    match = re.match(r'^(\d{1,3})([A-Z][A-Za-z].*)', line)
```

### 3. Markdown Citation Cleaning (`converter.py`)

The `_clean_markdown_citations()` method handles the critical conversion:

```python
def _clean_markdown_citations(self, text: str, citations: List[Dict]) -> str:
    """
    Clean up markdown text by:
    1. Converting inline citation refs (e.g., "text.10 11 12" -> "text. [^10] [^11] [^12] ")
    2. Removing footnote definition lines from body text
    """

    # Key challenges solved:
    # - Distinguish decimals ($1.5 billion) from citations (text.15)
    # - Handle citation clusters (10 11 12) vs single refs (15)
    # - Remove footnote definitions that appear inline in LaTeX PDFs
    # - Proper spacing around citations
```

**Decimal vs Citation Detection:**

```python
def replace_end_of_sentence_refs(match):
    before = match.group(1) or ""  # Character before punct
    punct = match.group(2)  # . or , or ; or :
    numbers_str = match.group(3)
    after = match.string[match.end():match.end()+20]

    ref_nums = re.findall(r'\d+', numbers_str)

    # Check if this looks like a decimal number
    if before and before.isdigit() and punct == '.':
        # Check for monetary/unit context
        if re.match(r'\s*(billion|million|thousand|percent|%)', after, re.IGNORECASE):
            return match.group(0)  # It's a decimal with units

        # Single digit after period is likely decimal
        if len(ref_nums) == 1 and int(ref_nums[0]) < 10:
            return match.group(0)  # Keep as decimal

    # Convert to citation format with proper spacing
    valid_refs = [n for n in ref_nums if n in known_refs]
    if valid_refs:
        refs_formatted = ' '.join(f'[^{n}]' for n in valid_refs)
        return (before or '') + punct + ' ' + refs_formatted + ' '
```

## CLI Usage

### Scraper CLI

```bash
# Parse a single PDF
python cli/parse_research_pdf.py path/to/research.pdf

# Parse to specific output directory
python cli/parse_research_pdf.py report.pdf -o output/parsed/

python cli/parse_research_pdf.py data/research/*.pdf -o output/parsed/
```

## Bug Fixes During Development

### 1. LaTeX Footnote Detection
- **Problem**: Position-based footnote detection failed because PyMuPDF extracts text linearly
- **Solution**: Added `extract_latex_footnotes()` for pattern-based detection of `\d+[A-Z]` patterns
- **Result**: Citation detection improved from 1 to 37 citations

### 2. Decimal vs Citation Confusion
- **Problem**: `$1.5 billion` was converting to `$1.[^5] billion`
- **Solution**: Added context checking for currency symbols, unit words (billion/million), and single-digit heuristics
- **Result**: Decimal numbers preserved correctly

### 3. Citation Spacing
- **Problem**: Citations rendered as `text.[^10][^11][^12]Text`
- **Solution**: Updated regex replacement to add proper spacing
- **Result**: Proper format: `text. [^10] [^11] [^12] Text`

### 4. Footnote Definitions in Body
- **Problem**: Footnote text like `10Martens CR, et al...` appearing in body content
- **Solution**: Added footnote definition detection and removal in `_clean_markdown_citations()`
- **Result**: Clean body text with definitions only in References section

## Known Limitations

2. **Complex tables**: Table extraction is basic
3. **Non-numeric citations**: Author-date style (Smith, 2024) not fully supported
4. **Missing citations**: Some refs may not be detected if pattern doesn't match
