# Changelog — 2026-06-14_01

## What this entry is

Six days of firm-onboarding work surfaced enough reusable orchestrator-level material — a new skill, two CLI tools, three issue write-ups, one spec, a small directory rename, and a CLAUDE.md discipline note — to log as a coherent batch. None of this entry describes what happened inside any firm-private `io/<firm>/` submodule. The deal content that surfaced these discoveries stays in its respective firm's own changelog.

## The pattern behind the batch

Setting up the first deal for a new firm (Alpha JWC Ventures) exposed every soft spot in the pipeline at once: deck input was a DocSend-gated PDF the orchestrator couldn't ingest, the firm's brand needed reverse-engineering from a heuristic that picked the wrong primary color, the firm's outline was a fork of an existing outline that needed section reordering, and several agents leaked context they shouldn't have (DocSend watermarks misread as the company name; the firm's geography bleeding into company-specific risk prose; citation enrichment fabricating example.com URLs even in codified mode). Each soft spot got either a fix that lives in code or a written diagnosis that lives in `context-v/issue-resolution/`. The new `setup-new-firm` skill is the consolidated procedure that captures what to do, in what order, and what to watch for — so the next firm doesn't get to rediscover all of this independently.

## What got built that's reusable across firms

**CLI: DocSend capture.** `cli/capture_docsend.py` drives a headless Chromium via Playwright to screenshot a DocSend deck page-by-page and assemble the screenshots into a PDF the existing `deck_analyst.py` agent can consume. Supports the two URL shapes DocSend ships (direct-share links that need no email gate, and standard links with `--email` gating). Refuses NDA-gated decks (eSignature required) on principle — the docstring explains that bypassing an NDA would be a contractual violation, and points the analyst at the manual-screenshot + PyMuPDF path for those. The companion `cli/_docsend_diag.py` is a diagnostic v2 with anti-WAF user-agent tweaks, form-scoped gate selectors, and a post-submit DOM dump for when the main script fails on a new DocSend page variant.

**Skill: setup-new-firm.** The five-phase onboarding procedure, written so an agent reading the SKILL.md can execute it without re-asking the analyst for context that's been settled. Phase boundaries are deliberate: brand acquisition is its own phase because Firecrawl's primary-color heuristic is wrong often enough that the analyst-override step is a hard requirement, not a fallback. The deck-input phase routes through the new CLI scripts above. The optional private-repo split phase uses `gh repo create --private` + `git submodule add` because that's the pattern every existing firm submodule was built with, and inventing a new pattern per firm would fragment the tree.

**Issue-resolution backlog.** Three write-ups, each naming a real pipeline gap surfaced by recent runs:

- *Competitive-Research-Generated-But-Not-In-Prose* — the competitive landscape evaluator and the writer agent produce/consume the same memo with no pipe between them. The fix is one block in the writer's outline preferred_sources pointing at the evaluator output file.
- *Including-Comparable-Exits-Valuations-IPOs* — Cash-on-Cash Return Probability has no dedicated source for exit comps, so the scenarios it renders are anchored to nothing the analyst can verify. The fix is a dedicated comps agent feeding a structured `2-tables/exit-comps.json`.
- *Limiting-or-Omitting-Investor-Judgement* — the pipeline emits PASS/CONSIDER/COMMIT verdicts at the end of two sections even when the analyst's job is advocacy. A `--no-verdict` flag (or a mode-level switch derived from `memo_mode: "justify"`) is the fix.

These three issues are now documented, not yet fixed. The fixes are independent and can be picked up in any order.

**Spec: Per-Deal Focal Points.** Analysts have opinions about a deck before the orchestrator sees it — founder's prior wins, partner-thesis match, IC concern to preempt, the angle that makes this deck worth the partnership's time. The orchestrator's prompts currently treat that input as "notes" — a soft, optional context block. The spec elevates it to a first-class input: a small ordered list the writer agents must respect as load-bearing emphasis. Implementation is deferred; the spec is the contract we'll build to.

## Directory rename: `context-v/skills/` → `context-v/agent-skills/`

The parent ai-labs/memopop-ai repo already uses `context-v/agent-skills/`. The orchestrator was using `context-v/skills/` for the same purpose. Renamed `adhoc-edits-workflow/SKILL.md` and `manage-memo-citations/SKILL.md` into the new path. No content changes. This is hygiene, not a refactor — keeps the convention consistent across the memopop tree so agents reading either repo's CLAUDE.md don't have to map between two names.

## CLAUDE.md: changelog scope discipline

A new "Changelog scope" section, inserted right after "Recent Changes (2025-11-20)" and before "Ongoing Troubleshooting." Draws the line clearly: orchestrator code/prompts/outlines/CLI/docs belong here; firm-private content (memo versions, brand config tweaks inside `io/<firm>/`, deal setups, source curation, fact-check corrections) belongs in the firm's own changelog inside its private submodule. Submodule pointer bumps get a one-line "Also included" entry, not a paragraph. Patterns discovered while editing client memos can land here as skills or issue-resolution entries — the *pattern* is reusable; the changelog entry references the skill, not the memo that surfaced it.

This rule existed in practice but wasn't written down. Today's entry is the first one fully respecting it (the original draft of this changelog described a Panthalassa-specific edit; that content has moved to the alpha-jwc submodule where it belongs).

## What didn't change

- No code in `src/`. The pipeline behavior is unchanged.
- No outline YAML, no template, no brand-config schema.
- No agent prompts.

This is a tooling + documentation batch. The fixes implied by the issue-resolution write-ups are deferred and will land as their own commits.

## What's next

- Wire `setup-new-firm` into the agent-skill discovery path so it auto-loads when an analyst says "set up a new firm."
- Build the Per-Deal Focal Points spec into the writer's prompt scaffold.
- The DocSend CLI scripts will get exercised against the next firm's first deck; expect at least one new DocSend page variant to break `capture_docsend.py` and require a `_docsend_diag.py` follow-up.
