# Release v0.5.1

**Investment Memo Orchestrator** — Multi-agent system for generating professional investment memos using LangGraph and Claude.

This release adds **two industry-standard memo outline templates** (Decile Group and Alpha Partners 7Cs) on top of the v0.5.0 improvements: LLM-powered fact-check pipeline, per-section source catalogs, TOC reliability fixes, and mechanical disambiguation enforcement. The pipeline stands at 31 agents with a focus on accuracy, traceability, and research artifact quality.

## Included from v0.5.0

### LLM-Powered Fact Verification & Correction

The fact-checker is upgraded from regex-only to a three-step pipeline:

```
fact_check (regex) → fact_verify (Perplexity) → fact_correct (Claude)
    Extract claims      Verify independently       Apply surgical fixes
```

- **fact_verifier**: Sends suspicious claims to Perplexity Sonar Pro. Tags each as confirmed / contradicted / corrected / unverifiable.
- **fact_corrector**: Uses Claude to surgically update specific sentences with verified corrections and new citations. Does NOT rewrite sections wholesale.
- **Traceability**: Full chain logged in `4-corrections-log.json` — old claim → finding → new claim → new citation.

### Per-Section Complete Source Lists

New `source_cataloger` agent compiles `3-source-catalog/{section}-Complete-Source-List.md` for each section, documenting every source the pipeline encountered: included, excluded, hallucinated, verified, and added-by-correction. Valuable for analysts who want to iterate on research beyond the final memo.

### Table of Contents Reliability (Fixed)

Two bugs prevented the TOC from appearing in final output:
1. TOC agent ran before the final draft file existed (workflow ordering)
2. `integrate_scorecard` always hit a fallback path that skipped TOC (missing `cli/__init__.py`)

Both fixed. TOC agent is now context-aware — validates accuracy and location, regenerates only if stale.

### Mechanical Disambiguation Enforcement

Prompt-based disambiguation was unreliable — Perplexity ignored exclusion instructions for Metabologic vs Metabolic across 4 versions. New mechanical override forces correct company URL and name after LLM synthesis, eliminating the entity mismatch cascade that tanked scores to 0%.

## Pipeline (31 nodes)

```
PHASE 1: DATA GATHERING
  dataroom → deck_analyst → research → section_research →
  competitive_researcher → competitive_evaluator

PHASE 2: CITATION ENRICHMENT ON RESEARCH
  cite → cleanup_research (GATE 1)

PHASE 3: WRITING
  draft

PHASE 4: SECTION ENRICHMENT
  inject_deck_images → enrich_trademark → enrich_socials →
  enrich_links → generate_tables → generate_diagrams →
  enrich_visualizations

PHASE 5: CONTENT REVISION
  revise_summaries

PHASE 6: PRE-ASSEMBLY CLEANUP
  cleanup_sections (GATE 2)

PHASE 7: FINAL DRAFT ASSEMBLY
  assemble_citations → toc

PHASE 8: VALIDATION & FACT-CHECK
  validate_citations → fact_check → fact_verify → fact_correct →
  source_catalog

PHASE 9: SCORING & FINALIZATION
  validate → scorecard → integrate_scorecard →
  finalize | human_review
```
