# Release v0.4.3

**Investment Memo Orchestrator** - Multi-agent system for generating professional investment memos using LangGraph and Claude.

This release delivers a **complete overhaul of deck slide classification and image placement**, a **fresh-start mode** for clean regeneration, **cap table extraction** as a new feature, and numerous bug fixes across the citation pipeline, link enrichment, and export system.

---

## Highlights

### Deck Slide Classification & Placement Overhaul (New in v0.4.3)

The deck screenshot system was misclassifying slides and spamming sections with duplicates. This release replaces the entire pipeline.

**Before:** 7 vague categories, 0.3x resolution for classification, many-to-many fanout placing the same image in 4+ sections, all images prepended at top of files.

**After:** 22 granular categories with a standalone classification guide, 0.5x resolution, one primary section per image, header-aware placement, max 2 placements per image, Key Slides gallery in Executive Summary.

| Old Classification | New Classification | Slide Content |
|---|---|---|
| `product` | `fundraising` | Pre-seed round status, use of funds |
| `product` | `solution` | Value proposition / mechanism of action |
| `product` | `branding` | Brand positioning, D2C website |
| `product` | `unit-economics` | Per-unit margins, COGS table |
| `product` | `technology` | AI platform track record, publications |

**New classification guide** (`templates/deck-classification-guide.md`):
- 22 categories with "use for", "not this category", visual signals, example slugs
- Suggested Placement in Memo for each category
- 12 ambiguity tiebreaker rules
- Section mapping tables for all 3 outline types
- 5 placement rules (max-2, one primary section, header-aware, Key Slides, excluded sections)

**New filename format:**
```
Old:  page-02-product.png
New:  page-02-fundraising-pre-seed-use-of-funds.png
```

### Fresh Start Mode (New since v0.4.2)

New CLI flags for clean regeneration:

```bash
# Ignore all prior artifacts, generate everything from scratch
python -m src.main "Company Name" --fresh

# Fresh start at a specific version
python -m src.main "Company Name" --fresh --version v0.1.0
```

### Cap Table Extraction (New since v0.4.2)

Initial feature for extracting structured information from cap table documents. Still requires iteration but provides a foundation for automated cap table analysis.

## Breaking Changes

- **`0-deck-analysis.json` format changed**: `section_to_screenshots` is now `{"images": [...], "key_slides": [...]}` instead of the old `{"keyword": ["path", ...]}` flat dict. The injection agent detects legacy format and prints a re-run message. Only affects runs that need screenshot injection from pre-v0.4.3 deck analysis output.
- **Screenshot filenames changed**: Old format `page-02-product.png` → new format `page-02-fundraising-pre-seed-use-of-funds.png`. Existing screenshots are not renamed; only new runs produce the new format.

## Installation

```bash
git clone https://github.com/lossless-group/investment-memo-orchestrator.git
cd investment-memo-orchestrator
git checkout v0.4.3

# Setup
uv venv --python python3.11
source .venv/bin/activate
uv pip install -e .

# Configure API keys
cp .env.example .env
# Edit .env with your ANTHROPIC_API_KEY, TAVILY_API_KEY, PERPLEXITY_API_KEY
```

---

## Requirements

- Python 3.11+
- `uv` package manager (NOT pip)
- API keys: Anthropic (required), Tavily (recommended), Perplexity (recommended)
- System dependencies: Pandoc, Poppler (for PDF screenshots), Cairo/Pango (for PDF export)
