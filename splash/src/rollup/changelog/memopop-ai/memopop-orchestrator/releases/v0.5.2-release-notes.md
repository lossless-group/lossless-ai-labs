# Release v0.5.2

**Investment Memo Orchestrator** — Multi-agent system for generating professional investment memos using LangGraph and Claude.

This release introduces the **One-Pager Generator** — a new pipeline agent and CLI tool that distills the full memo into a branded single-page visual summary designed by Claude. Also includes a **citation spacing agent** for consistent markdown formatting and **branding fixes** for dark mode and header logo rendering.

---

## Highlights

### One-Pager Generator (New Feature)

A two-step LLM pipeline that produces a polished, single-page investment summary:

**Step 1 — Content Extraction**: Claude reads the full memo and extracts content into structured JSON slots: a punchy headline lede (one sentence), extended context, 3 key bullets, deal terms, market stats, competitive positioning, traction milestones, and recommendation.

**Step 2 — Layout Design**: Claude receives the brand config (colors, fonts, logos) and content slots, and generates a complete self-contained HTML page with CSS grid layout. The design is fresh each time — no fixed template.

```
Full Memo (7-*.md) → Extract Slots (Claude) → Design Layout (Claude) → PDF (WeasyPrint)
```

**Framing**: The one-pager reads as "[VC Firm] is syndicating an investment in [Company]" — the VC firm logo appears at the top of the sidebar as the presenter, the company logo below as the subject.

**Split headline**: The lede is one punchy sentence (~12pt bold, max 2 lines) followed by supporting context in smaller text. Prevents the wall-of-bold-text problem.

**Mechanical post-processing**: Logos are injected via `%%PLACEHOLDER%%` markers (not trusted to Claude), company names are disambiguated, and duplicate logos are removed. This belt-and-suspenders approach handles the inherent unreliability of LLM-generated HTML.

**Output artifacts**:
```
output/{Deal}-{Version}/
├── 8-one-pager.html          # Standalone HTML (scrollable in browser)
├── 8-one-pager.pdf           # Single-page PDF (US Letter)
└── 8-one-pager-content.json  # Extracted content slots
```

**CLI for standalone use**:
```bash
python -m cli.generate_one_pager "Metabologic" --firm humain
python -m cli.generate_one_pager "Metabologic" --firm humain --mode dark
python -m cli.generate_one_pager "Metabologic" --dry-run  # preview slots only
```

**Pipeline position**: Runs after `integrate_scorecard`, before `finalize`.

### Citation Spacing Agent (New)

Purely mechanical agent (no LLM, regex only) that enforces correct markdown citation spacing after final draft assembly. Fixes five recurring issues:

| Rule | Before | After |
|------|--------|-------|
| Space after punctuation | `claim.[^1]` | `claim. [^1]` |
| Space between consecutive | `[^1][^2]` | `[^1] [^2]` |
| Deduplicate consecutive | `[^3] [^3]` | `[^3]` |
| Definitions at column 0 | `  [^1]: text` | `[^1]: text` |
| Space after colon | `[^1]:text` | `[^1]: text` |

Runs after `toc`, before `validate_citations`. Tested on Metabologic v0.2.4: 12 fixes applied.

### Branding Fixes

- **Dark mode text**: All dark mode CSS rules in `base-style.css` were using `var(--brand-background)` as text color (the background color, not text). Fixed to use `var(--brand-text-light)`.
- **Header logo**: The memo header always has a dark background, so it always needs the white/light logo. Fixed logo selection to use `dark_mode` logo regardless of export mode.
- **Header children**: Added `.memo-header` scoped CSS overrides so title, subtitle, and meta labels inherit light text from the header instead of using body-level dark colors.
